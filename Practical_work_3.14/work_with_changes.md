## Работа с изменениями

<span style="color:red">**git add *[файл]*** </span>- добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита. По умолчанию git commit использует лишь этот индекс, так что вы можете использовать git add для сборки слепка вашего следующего коммита.

Пример транскрипции :

```bash=
git add
```
Есть похожие команды, например, 
```bash=
git add .
``` 
индексирует сразу все изменённые файлы и папки в директории, где вы находитесь. Обратите внимание, между точкой и словом add нужно ставить пробел. 

Команда
```bash=
git add :/ 
```
добавляет в индекс все файлы независимо от того, в какой директории вы находитесь.

<span style="color:red">**git commit *[Комментарий_к_коммиту]***</span> - cделать коммит. При выполнении команды git commit изменения всех файлов, внесённые в индекс, переносятся в репозиторий. При создании коммита (точки фиксации изменений) требуется указать сообщение. Параметр -m позволяет написать сообщение из командной строки.

Пример транскрипции:

```bash=
git commit -m
```

<span style="color:red">**git log**</span> - досмотреть историю коммитов. Выводит список всех коммитов. У этой команды есть разные опции, самая используемая из них —
```bash=
--oneline
``` 
Она показывает хеш в укороченном формате, ветку, в которой сделан коммит, а также текст коммита. Чтобы использовать эту опцию (как и любую другую), нужно добавить её после команды.

Пример транскрипции:

```bash=
git log--oneline
```

<span style="color:red">**git pull** </span> - подтянуть изменения. Подтягивает в локальный репозиторий последнюю версию проекта. Вызов этой команды сотрёт все незафиксированные изменения.

Пример транскрипции:

```bash=
git pull
```

<span style="color:red">**git push** </span> - отправляет все зафиксированные изменения с локального репозитория в удалённый. Это одна из самых важных команд, ведь все вышеописанные действия производятся в локальной копии репозитория. Когда вы закончите работу, эту копию нужно будет отправить в удалённый репозиторий. Только так другие участники процесса смогут получить актуальную версию.

Пример транскрипции:

```bash=
git push
```

<span style="color:red">**git status** </span> - посмотреть статус файлов. Вы увидите, какие файлы изменили, удалили или добавили в проект.

Пример транскрипции:

```bash=
git status
```
