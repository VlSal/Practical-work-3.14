## Работа с ветками

<span style="color:red">**git branch** </span>- посмотреть все локальные ветки.

Пример транскрипции:

```bash=
git branch
```

<span style="color:red">**git branch -m *[Старое_название_ветки] [Новое_название_ветки]***</span> - переименовывает ветку. Названия нужно писать на латинице.

Пример транскрипции:

```bash=
git branch -m [Старое_название_ветки] [Новое_название_ветки]
```

<span style="color:red">**git branch --delete *[Название_ветки]***</span> - команда удаляет ветку [Название_ветки] в локальном репозитории. Если нужно избавиться от ветки в удалённом репозитории, используйте 
```bash=
git push --delete origin [name].
```
Пример транскрипции:

```bash=
git branch --delete [name]

<span style="color:red">**git cherry-pick**</span> - cоздать точную копию коммитов. Команду часто совмещают с git merge и git rebase, чтобы сохранить линейную историю коммитов. То есть создаётся точная копия коммитов, выполняется перебазирование и слияние веток.

Пример транскрипции:

```bash=
git cherry-pick
```

<span style="color:red">**git merge *[Название_ветки]*** </span> - влить ветку. Вливает ветку "Название ветки" в ветку, в которой вы находитесь.

Пример транскрипции:

```bash=
git merge [name]
```

<span style="color:red">**git push origin *[Название_ветки]*** </span> - добавляет новую ветку с названием "Название ветки" и автоматически переключает на неё.

Пример транскрипции:

```bash=
git push origin [name]
```

<span style="color:red">**git merge *[Название_ветки]*** </span> - перебазировать коммиты. Перебазирует коммиты из ветки, в которой вы находитесь, в ветку "Название ветки".

Пример транскрипции:

```bash=
git rebase [name]
```

<span style="color:red">**git switch --create *[Название_ветки]*** </span>- создать ветку. Добавляет новую ветку с названием "Название ветки" и автоматически переключает на неё.

Пример транскрипции:

```bash=
git switch --create [name]
```

<span style="color:red">**git switch *[Название_ветки]*** </span>- переключить ветку. Вы перейдёте на уже созданную ветку "Название ветки".

Пример транскрипции:

```bash=
git switch [name]
```
Для создания и переключения веток также можно использовать:
```bash=
git checkout [master]
```
Эта команда появилась раньше, у неё есть множество дополнительных функций. Например, она может восстанавливать изменения в коммите. Как раз из-за такого разнообразия задач разработчики решили создать отдельную команду для переключения между ветками — `git switch`.
